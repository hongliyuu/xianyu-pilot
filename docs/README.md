# 文档索引

本目录是项目文档的统一入口。每个主题只维护一份权威文档；其他文档应链接到
该文档，而不是复制操作步骤。

| 目标 | 文档 | 唯一事实来源 |

## 文档规则

- 操作步骤必须保留在其所有者文档中，其他位置只链接。
- 修改行为、部署、安全边界或运维命令时，必须在同一变更中更新对应文档。
- 计划工作、未验证流程和历史记录必须明确标识，不能表述为已经完成的生产证据。
- 命令必须能在标注的 Shell 中直接执行，且不得通过命令参数传递密钥。
- 根目录 `README.md` 只保留产品概览和入口；运维细节归入本目录的专题文档。

## 文档生命周期

1. 在依赖某项技术决策或运维流程前，先将其写入对应的所有者文档。
2. 根 README 和本索引仅承担导航职责，不复制专题操作手册。
3. Pull Request 中说明已更新的所有者文档；如无需更新，明确理由。
4. 发布前，结合本文档索引、生产就绪清单和变更日志一起评审。
